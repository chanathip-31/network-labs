RSTP (Rapid Spanning Tree Protocol)
Rapid Spanning Tree Protocol คือ STP เวอร์ชันที่เร็วขึ้น ใช้ป้องกัน Loop ในระบบ Switch Layer 2
ถ้าเราต่อ Switch แบบมี “เส้นสำรอง” จะเกิดปัญหา Broadcast Loop ได้ เช่น

SW1 ----- SW2
 |         |
 |         |
SW3 -------

เฟรมจะวนไม่จบ ทำให้:
	- Network ช้า
	- CPU Switch พุ่ง
	- MAC Address Table เพี้ยน
	- ใช้งานไม่ได้ทั้งวง
RSTP จึงช่วย:
	- ป้องกัน Loop
	- เลือกเส้นทางหลัก
	- ปิดเส้นสำรองชั่วคราว
	- ถ้าเส้นหลักล่ม → สลับไปเส้นสำรองเร็วมาก (1-3 วินาที)
 
 หลักการทำงานของ RSTP
1. Root Bridge Switch หลักของระบบ
	- Priority ต่ำสุด
	- ถ้าเท่ากันดู MAC ต่ำสุด
2. Root Port (RP)
	- Port ที่วิ่งไปหา Root Bridge ดีที่สุด มีได้ 1 Port ต่อ Switch
3. Designated Port (DP)
	- Port หลักของ Segment นั้น
4. Alternate Port 
	- Port สำรอง ปกติจะ Block ไว้ ถ้าเส้นหลักล่ม → เปลี่ยนเป็น Forwarding ทันที

STP กับ RSTP ต่างกันยังไง
| STP                      	| RSTP        
| convergence ช้า 30-50 วิ 	| เร็ว 1-3 วิ 
| port state เยอะ          	| state น้อย   
| ไม่มี Alternate Port     	| มี          
| ใช้ 802.1D               	| ใช้ 802.1w   

Port State ของ RSTP
| State      | ความหมาย        
| Discarding | ยังไม่ส่งข้อมูล 	   
| Learning   | เรียนรู้ MAC       
| Forwarding | ส่งข้อมูลได้        

Topology 

        SW1
       /   \
      /     \
    SW2-----SW3
	
Loop แน่นอน → RSTP จะ Block 1 Port

เป้าหมายของ Lab
	- เปิด RSTP
	- ดู Root Bridge
	- ดู Port Role
	- Port State
	- ทดลองดึงสายดู Failover

ขั้นตอนทำ Lab
STEP 1 — ต่อสาย
SW1 Fa0/1 <-> SW2 Fa0/1
SW1 Fa0/2 <-> SW3 Fa0/1
SW2 Fa0/2 <-> SW3 Fa0/2

>> STEP 2 — เปิด RSTP ทำทุกตัว
พิมพ์ CLI:
spanning-tree mode rapid-pvst
ตรวจสอบ พิมพ์ CLI:
show spanning-tree summary >> ควรเห็น Switch is in rapid-pvst mode

>> STEP 3 — ตั้ง Root Bridge ให้ SW1 เป็น Root
พิมพ์ CLI บน SW1:
spanning-tree vlan 1 root primary
ตรวจสอบ พิมพ์ CLI:
show spanning-tree >> จะเห็น This bridge is the root 

>> STEP 4 — ตรวจสอบ Port Role บน SW2/SW3
พิมพ์ CLI บน SW2/SW3
show spanning-tree
จะเห็นประมาณนี้:
Role Sts Cost Prio.Nbr Type
Root FWD
Desg FWD
Altn DISC

>> STEP 5 — ทดลอง Failover
ปิด Port ระหว่าง SW1 กับ SW2
พิมพ์ CLI บน SW1 หรือ SW2:
interface fa0/0
shutdown
RSTP จะ:
	- เอา Alternate Port ขึ้นมาแทน
	- Network ยังวิ่งต่อได้
เร็วกว่า STP มาก
